# Runbooks

Операционные инструкции — точные последовательности команд для повторяющихся задач: deploy на новую сеть, verify контрактов, миграция registry, ротация ключей, recovery после инцидента.

## Принципы

- **Один runbook — одна задача.** Не пихаем deploy и migration в один файл.
- **Точные команды.** Вся команда копипастится как есть. Нет «подставьте значение» без примера.
- **Pre-flight check.** В начале runbook — чек-лист: что должно быть готово до запуска.
- **Post-flight verification.** В конце — как убедиться, что всё сработало.
- **Rollback.** Если применимо — отдельный раздел «как откатить».
